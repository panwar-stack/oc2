oc2 можно запускать напрямую в Windows, но для лучшего опыта мы рекомендуем [Windows Subsystem for Linux (WSL)](https://learn.microsoft.com/en-us/windows/wsl/install). WSL дает Linux-среду, которая отлично работает с возможностями oc2.

:::tip[Почему WSL?]
WSL дает более высокую производительность файловой системы, полноценную поддержку терминала и совместимость с инструментами разработки, на которые опирается oc2.
:::

---

## Настройка

<Steps>

1. **Установите WSL**

   Если вы еще не сделали этого, установите WSL по [официальному руководству Microsoft](https://learn.microsoft.com/en-us/windows/wsl/install).

2. **Установите oc2 в WSL**

   После настройки WSL откройте терминал WSL и установите oc2 одним из [способов установки](/docs/).

   ```bash
   curl -fsSL https://oc2.ai/install | bash
   ```

3. **Запускайте oc2 из WSL**

   Перейдите в каталог проекта (к файлам Windows можно обращаться через `/mnt/c/`, `/mnt/d/` и т.д.) и запустите oc2.

   ```bash
   cd /mnt/c/Users/YourName/project
   oc2
   ```

</Steps>

## Десктопное приложение + сервер в WSL

Если вы предпочитаете oc2 Desktop, но хотите запускать сервер в WSL:

1. **Запустите сервер в WSL** с параметром `--hostname 0.0.0.0`, чтобы разрешить внешние подключения:

   ```bash
   oc2 serve --hostname 0.0.0.0 --port 4096
   ```

2. **Подключите десктопное приложение** к `http://localhost:4096`

:::note
Если в вашей конфигурации `localhost` не работает, используйте IP-адрес WSL (выполните в WSL: `hostname -I`) и подключайтесь по `http://<wsl-ip>:4096`.
:::

:::caution
При использовании `--hostname 0.0.0.0` задайте `OC2_SERVER_PASSWORD`, чтобы защитить сервер.

```bash
OC2_SERVER_PASSWORD=your-password oc2 serve --hostname 0.0.0.0
```

:::

## Веб-клиент + WSL

Для лучшего веб-опыта в Windows:

1. **Запускайте `oc2 web` в терминале WSL**, а не в PowerShell:

   ```bash
   oc2 web --hostname 0.0.0.0
   ```

2. **Открывайте в браузере Windows** адрес `http://localhost:<port>` (oc2 выведет URL)

Запуск `oc2 web` из WSL обеспечивает корректный доступ к файловой системе и интеграцию с терминалом, при этом интерфейс остается доступным из браузера Windows.

## Доступ к файлам Windows

WSL может получать доступ ко всем вашим файлам Windows через каталог `/mnt/`:

- `C:` drive → `/mnt/c/`
- `D:` drive → `/mnt/d/`
- И так далее

Пример:

```bash
cd /mnt/c/Users/YourName/Documents/project
oc2
```

:::tip
Для максимально плавной работы стоит клонировать или скопировать репозиторий в файловую систему WSL (например, в `~/code/`) и запускать oc2 оттуда.
:::

---

## Советы

- Даже для проектов на дисках Windows запускайте oc2 в WSL, чтобы получить более плавный доступ к файлам
- Используйте oc2 вместе с [расширением WSL для VS Code](https://code.visualstudio.com/docs/remote/wsl) для единого рабочего процесса
- Конфигурация и сессии oc2 хранятся в среде WSL по пути `~/.local/share/oc2/`
