---
title: Supervisor
description: Configure session supervision and visible steering recommendations.
---

The supervisor is an opt-in reviewer for agentic coding sessions. It watches observable session artifacts, derives compact progress and risk state, and can ask a separate supervisor model to recommend a next action when the primary coding session appears to drift.

The first implementation is in-process. It covers crawl and early walk behavior: session observation, risk state, API/SSE surfaces, per-session settings, continuous model review at meaningful boundaries, after-action reports, and visible recommendation insertions. It does not pause, block, fork, roll back, deny permissions, or enforce policy.

---

## Configure

Set global defaults in `opencode.json` with the `supervisor` key.

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "supervisor": {
    "mode": "observe",
    "recommendation_model": "anthropic/claude-sonnet-4-5",
    "recommendation_variant": "fast",
    "review_cadence": "step",
    "insert_recommendations": true
  }
}
```

`recommendation_model` is independent from the primary session model, and `recommendation_variant` selects an optional variant for that model. This lets you run a stronger or higher-quantized supervisor over a faster or cheaper coding model. If omitted, OpenCode resolves the supervisor model from top-level `model`, then `small_model`; if neither is available, it can still emit state and risks but will not create model recommendations.

[See all config keys and defaults](/docs/config#supervisor).

---

## Modes

- `off` - The effective session does not run active supervision. New reviews and recommendations are ignored or dropped.
- `observe` - OpenCode derives supervisor state and risks and emits `supervisor.state.updated`, but does not create recommendations.
- `advise` - OpenCode derives state, asks the supervisor model for bounded recommendations, emits recommendation events, and inserts accepted recommendations when `insert_recommendations` is enabled.

The effective mode is resolved from built-in defaults, global config, and then session overrides. Session overrides win over global config, so `/supervisor` can enable or disable supervision for one session without editing `opencode.json`.

---

## Session Control

Use `/supervisor` inside an active TUI session to inspect and change that session's supervisor settings.

```text
/supervisor
```

The dialog shows the effective mode, whether values come from global config or a session override, and the resolved recommendation model. It can set `off`, `observe`, or `advise`, toggle transcript insertion, edit runtime limits, and reset session overrides.

`/supervisor` is a local TUI command. It is not a model-facing slash command and is not sent through the session prompt command path.

---

## Review Boundary

The supervisor reviews meaningful state changes, not every streamed token. Meaningful boundaries include finished steps, completed or failed tool calls, changed diffs, permission requests, idle-after-edit transitions, and deterministic risk triggers.

The supervisor stays silent when progress looks acceptable. It should recommend only when bounded state indicates likely drift, missing validation, wrong localization, evidence mismatch, premature success, or a clearly less optimal next action.

---

## Privacy Boundary

The supervisor uses observable artifacts only: visible prompt text summaries, message parts, tool calls, command status, permission events, diffs, and validation outcomes.

The recommendation model receives bounded, redacted state. It does not receive raw command output, full file contents, private reasoning, or the full user prompt text. Only the accepted recommendation text is inserted into the primary agent conversation; raw supervisor state, reports, evidence internals, command output, and file contents stay outside the primary model context.

---

## Transcript Insertions

In `advise` mode, accepted recommendations can be inserted as visible supervisor-authored conversation content. Insertions are synthetic user text parts marked with supervisor metadata, visible in the transcript, and included in the next primary model input.

OpenCode re-checks the latest effective mode and `insert_recommendations` setting immediately before insertion. If the session switched to `off` or insertion was disabled while review was in flight, queued content is dropped.

Example insertion:

```text
Supervisor recommendation: missing_validation

Run a relevant validation command before marking this complete.

Evidence:
- Edited files: packages/opencode/src/session/session.ts
- No successful validation command after the first patch
```

Recommendations are bounded by `max_recommendation_chars` and `max_recommendations_per_session`.

## API

Supervisor endpoints are session-scoped.

| Method | Path | Description |
| --- | --- | --- |
| `GET` | `/session/:sessionID/supervisor` | Get the effective supervisor config and current derived state. |
| `PATCH` | `/session/:sessionID/supervisor` | Update durable per-session supervisor overrides. |
| `GET` | `/session/:sessionID/supervisor/report` | Generate and return an after-action supervisor report. |

Settings patch fields are optional. Omitted fields keep their current value, `null` clears that session override key, and `reset: true` clears all session overrides before applying any non-null fields in the same request.

```json title="PATCH /session/:sessionID/supervisor"
{
  "mode": "advise",
  "insert_recommendations": true,
  "recommendation_model": "anthropic/claude-sonnet-4-5",
  "recommendation_variant": "fast"
}
```

```json title="Clear one override"
{
  "recommendation_model": null,
  "recommendation_variant": null
}
```

```json title="Reset session overrides"
{
  "reset": true
}
```

## SSE Events

The normal `/event` and `/global/event` streams include supervisor events.

| Event | Description |
| --- | --- |
| `supervisor.state.updated` | Derived supervisor state changed for a session. |
| `supervisor.recommendation.created` | A model recommendation was accepted. |
| `supervisor.settings.updated` | Durable per-session supervisor settings changed. |
| `supervisor.report.completed` | An after-action supervisor report was generated. |

If an SSE client disconnects, refetch `GET /session/:sessionID/supervisor` to recover the latest state.

## SDK

The v2 SDK exposes the supervisor surface under `client.supervisor`.

```ts
const state = await client.supervisor.get({
  sessionID,
})

await client.supervisor.update({
  sessionID,
  supervisorSettingsPatch: {
    mode: "advise",
    insert_recommendations: true,
  },
})

const report = await client.supervisor.report({
  sessionID,
})
```
