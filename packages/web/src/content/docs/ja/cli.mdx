OC2 CLI は、引数なしで実行すると、デフォルトで [TUI](/docs/tui) を開始します。

```bash
oc2
```

ただし、このページに記載されているようにコマンドも受け入れます。これにより、OC2 をプログラム的に操作できるようになります。

```bash
oc2 run "Explain how closures work in JavaScript"
```

---

### tui

OC2 ターミナルユーザーインターフェイスを開始します。

```bash
oc2 [project]
```

#### フラグ

| フラグ                                   | ショート                                | 説明                                                       |
| ---------------------------------------- | --------------------------------------- | ---------------------------------------------------------- |
| <nobr><code>{"--continue"}</code></nobr> | `-c`                                    | 最後のセッションを続行                                     |
| <nobr><code>{"--session"}</code></nobr>  |                                         | 続行時にセッションをフォーク (`-s` または `--fork` と併用) |
| <nobr><code>{"--continue"}</code></nobr> | <nobr><code>{"--session"}</code></nobr> | 続行するセッション ID                                      |
| <nobr><code>{"--prompt"}</code></nobr>   |                                         | 使用のプロンプト                                           |
| <nobr><code>{"--model"}</code></nobr>    | `-m`                                    | プロバイダー/モデルの形式で使用するモデル                  |
| <nobr><code>{"--agent"}</code></nobr>    |                                         | 使用するエージェント                                       |
| <nobr><code>{"--port"}</code></nobr>     |                                         | リッスンするポート                                         |
| <nobr><code>{"--hostname"}</code></nobr> |                                         | リッスンするホスト名                                       |

## コマンド

OC2 CLI には次のコマンドもあります。

---

### agent

OC2 のエージェントを管理します。

```bash
oc2 agent [command]
```

---

### attach

`serve` または `web` コマンドを使用して起動された、すでに実行中の OC2 バックエンドサーバーにターミナルを接続します。

```bash
oc2 attach [url]
```

これにより、リモート OC2 バックエンドで TUI を使用できるようになります。例えば：

```bash
# Start the backend server for web/mobile access
oc2 web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
oc2 attach http://10.20.30.40:4096
```

#### フラグ

| フラグ                                   | ショート | 説明                                                                              |
| ---------------------------------------- | -------- | --------------------------------------------------------------------------------- |
| <nobr><code>{"--dir"}</code></nobr>      |          | TUI を開始する作業ディレクトリ                                                    |
| <nobr><code>{"--continue"}</code></nobr> | `-c`     | 最後のセッションを続行                                                            |
| <nobr><code>{"--session"}</code></nobr>  | `-s`     | 続行するセッション ID                                                             |
| <nobr><code>{"--fork"}</code></nobr>     |          | 続行時にセッションをフォーク（`--continue` または `--session` と使用）            |
| <nobr><code>{"--password"}</code></nobr> | `-p`     | Basic 認証パスワード（デフォルトは `OC2_SERVER_PASSWORD`）                   |
| <nobr><code>{"--username"}</code></nobr> | `-u`     | Basic 認証ユーザー名（デフォルトは `OC2_SERVER_USERNAME` または `oc2`） |

#### create

カスタム構成で新しいエージェントを作成します。

```bash
oc2 agent create
```

このコマンドは、カスタムシステムプロンプトとツール構成を使用して新しいエージェントを作成する手順を示します。

#### list

利用可能なエージェントをすべてリストします。

```bash
oc2 agent list
```

### auth

プロバイダーの資格情報とログインを管理するコマンド。

```bash
oc2 auth [command]
```

---

#### login

OC2 は [Models.dev](https://models.dev) のプロバイダーリストを利用しているため、`oc2 auth login` を使用して、使用したいプロバイダーの API キーを構成できます。これは `~/.local/share/oc2/auth.json` に保存されます。

```bash
oc2 auth login
```

OC2 が起動すると、認証情報ファイルからプロバイダーがロードされます。また、環境またはプロジェクト内の `.env` ファイルで定義されたキーがあるかどうかも確認します。

---

#### list

認証情報ファイルに保存されているすべての認証されたプロバイダーをリストします。

```bash
oc2 auth list
```

またはショートバージョン。

```bash
oc2 auth ls
```

---

#### logout

資格情報ファイルからプロバイダーをクリアすることで、プロバイダーからログアウトします。

```bash
oc2 auth logout
```

### github

リポジトリ自動化のための GitHub エージェントを管理します。

```bash
oc2 github [command]
```

#### install

GitHub エージェントをリポジトリにインストールします。

```bash
oc2 github install
```

これにより、必要な GitHub Actions ワークフローが設定され、構成プロセスがガイドされます。 [詳細はこちら](/docs/github)。

#### run

GitHub エージェントを実行します。これは通常、GitHub Actions で使用されます。

```bash
oc2 github run
```

##### フラグ

| フラグ                                | 説明                                                |
| ------------------------------------- | --------------------------------------------------- |
| <nobr><code>{"--event"}</code></nobr> | エージェントを実行するための GitHub モック イベント |
| <nobr><code>{"--token"}</code></nobr> | GitHub 個人アクセストークン                         |

### mcp

モデルコンテキストプロトコルサーバーを管理します。

```bash
oc2 mcp [command]
```

---

#### add

MCP サーバーを構成に追加します。

```bash
oc2 mcp add
```

このコマンドは、ローカルまたはリモートの MCP サーバーを追加する手順を示します。

---

#### list

構成されているすべての MCP サーバーとその接続ステータスをリストします。

```bash
oc2 mcp list
```

または、短いバージョンを使用してください。

```bash
oc2 mcp ls
```

---

#### auth

OAuth 対応の MCP サーバーで認証します。

```bash
oc2 mcp auth [name]
```

サーバー名を指定しない場合は、利用可能な OAuth 対応サーバーから選択するように求められます。

OAuth 対応サーバーとその認証ステータスを一覧表示することもできます。

```bash
oc2 mcp auth list
```

または、短いバージョンを使用してください。

```bash
oc2 mcp auth ls
```

---

#### logout

MCP サーバーの OAuth 資格情報を削除します。

```bash
oc2 mcp logout [name]
```

---

#### debug

MCP サーバーの OAuth 接続の問題をデバッグします。

```bash
oc2 mcp debug <name>
```

### models

構成されたプロバイダーから利用可能なすべてのモデルをリストします。

```bash
oc2 models [provider]
```

このコマンドは、構成されたプロバイダー全体で利用可能なすべてのモデルを `provider/model` の形式で表示します。

これは、[設定](/docs/config/) で使用する正確なモデル名を把握するのに役立ちます。

オプションでプロバイダー ID を渡して、そのプロバイダーによってモデルをフィルターできます。

```bash
oc2 models anthropic
```

#### フラグ

| フラグ                                  | 説明                                                            |
| --------------------------------------- | --------------------------------------------------------------- |
| <nobr><code>{"--refresh"}</code></nobr> | models.dev からモデルキャッシュを更新します。                   |
| <nobr><code>{"--verbose"}</code></nobr> | より詳細なモデル出力を使用します (コストなどのメタデータを含む) |

`--refresh` フラグを使用して、キャッシュされたモデルリストを更新します。これは、新しいモデルがプロバイダーに追加され、それを OC2 で確認したい場合に便利です。

```bash
oc2 models --refresh
```

### run

プロンプトを直接渡して、非対話モードで oc2 を実行します。

```bash
oc2 run [message..]
```

これは、スクリプト作成、自動化、または完全な TUI を起動せずに迅速な回答が必要な場合に便利です。例えば。

```bash "oc2 run"
oc2 run Explain the use of context in Go
```

実行中の `oc2 serve` インスタンスにアタッチして、実行ごとの MCP サーバーのコールドブート時間を回避することもできます。

```bash
# Start a headless server in one terminal
oc2 serve

# In another terminal, run commands that attach to it
oc2 run --attach http://localhost:4096 "Explain async/await in JavaScript"
```

#### フラグ

| フラグ                                   | ショート | 説明                                                                                      |
| ---------------------------------------- | -------- | ----------------------------------------------------------------------------------------- |
| <nobr><code>{"--command"}</code></nobr>  |          | 実行するコマンド。引数には message を使用します。                                         |
| <nobr><code>{"--continue"}</code></nobr> | `-c`     | 最後のセッションを続行                                                                    |
| <nobr><code>{"--session"}</code></nobr>  | `-s`     | 続行するセッション ID                                                                     |
| <nobr><code>{"--fork"}</code></nobr>     |          | 続行時にセッションをフォーク (`--continue` または `--session` と併用)                     |
| <nobr><code>{"--share"}</code></nobr>    |          | セッションを共有する                                                                      |
| <nobr><code>{"--model"}</code></nobr>    | `-m`     | プロバイダー/モデルの形式で使用するモデル                                                 |
| <nobr><code>{"--agent"}</code></nobr>    |          | 使用するエージェント                                                                      |
| <nobr><code>{"--file"}</code></nobr>     | `-f`     | メッセージに添付するファイル                                                              |
| <nobr><code>{"--format"}</code></nobr>   |          | 形式: デフォルト (フォーマット済み) または json (生の JSON イベント)                      |
| <nobr><code>{"--title"}</code></nobr>    |          | セッションのタイトル (値が指定されていない場合は、切り詰められたプロンプトが使用されます) |
| <nobr><code>{"--attach"}</code></nobr>   |          | 実行中の oc2 サーバー (http://localhost:4096 など) に接続します。                    |
| <nobr><code>{"--password"}</code></nobr> | `-p`     | Basic 認証パスワード（デフォルトは `OC2_SERVER_PASSWORD`）                           |
| <nobr><code>{"--username"}</code></nobr> | `-u`     | Basic 認証ユーザー名（デフォルトは `OC2_SERVER_USERNAME` または `oc2`）         |
| <nobr><code>{"--dir"}</code></nobr>      |          | 実行ディレクトリ、またはアタッチ時のリモートサーバー上のパス                              |
| <nobr><code>{"--variant"}</code></nobr>  |          | モデルバリアント（プロバイダー固有の推論レベル）                                          |
| <nobr><code>{"--thinking"}</code></nobr> |          | 思考ブロックを表示                                                                        |
| <nobr><code>{"--port"}</code></nobr>     |          | ローカルサーバーのポート (デフォルトはランダムポート)                                     |

### serve

API アクセスのためにヘッドレス OC2 サーバーを起動します。完全な HTTP インターフェイスについては、[サーバーのドキュメント](/docs/server) を確認してください。

```bash
oc2 serve
```

これにより、TUI インターフェイスを使用せずに oc2 機能への API アクセスを提供する HTTP サーバーが起動します。 `OC2_SERVER_PASSWORD` を設定して HTTP 基本認証を有効にします (ユーザー名のデフォルトは `oc2`)。

#### フラグ

| フラグ                                   | 説明                                    |
| ---------------------------------------- | --------------------------------------- |
| <nobr><code>{"--port"}</code></nobr>     | リッスンするポート                      |
| <nobr><code>{"--hostname"}</code></nobr> | リッスンするホスト名                    |
| <nobr><code>{"--mdns"}</code></nobr>     | mDNS 検出を有効にする                   |
| <nobr><code>{"--cors"}</code></nobr>     | CORS を許可する追加のブラウザーオリジン |

### stats

OC2 セッションのトークンの使用状況とコストの統計を表示します。

```bash
oc2 stats
```

#### フラグ

| フラグ                                  | 説明                                                                                       |
| --------------------------------------- | ------------------------------------------------------------------------------------------ |
| <nobr><code>{"--days"}</code></nobr>    | 過去 N 日間の統計を表示 (全期間)                                                           |
| <nobr><code>{"--tools"}</code></nobr>   | 表示するツールの数 (すべて)                                                                |
| <nobr><code>{"--models"}</code></nobr>  | モデルの使用状況の内訳を表示 (デフォルトでは非表示)。上位 N 件を表示するには数値を渡します |
| <nobr><code>{"--project"}</code></nobr> | プロジェクトでフィルタリング (全プロジェクト、空文字列: 現在のプロジェクト)                |

---

### export

セッションデータを JSON としてエクスポートします。

```bash
oc2 export [sessionID]
```

セッション ID を指定しない場合は、利用可能なセッションから選択するように求められます。

---

### import

JSON ファイルまたは OC2 共有 URL からセッションデータをインポートします。

```bash
oc2 import <file>
```

ローカルファイルまたは OC2 共有 URL からインポートできます。

```bash
oc2 import session.json
oc2 import https://opncd.ai/s/abc123
```

### web

Web インターフェイスを使用してヘッドレス OC2 サーバーを起動します。

```bash
oc2 web
```

これにより、HTTP サーバーが起動し、Web ブラウザが開き、Web インターフェイスを通じて OC2 にアクセスします。 `OC2_SERVER_PASSWORD` を設定して HTTP 基本認証を有効にします (ユーザー名のデフォルトは `oc2`)。

#### フラグ

| フラグ                                   | 説明                                    |
| ---------------------------------------- | --------------------------------------- |
| <nobr><code>{"--port"}</code></nobr>     | リッスンするポート                      |
| <nobr><code>{"--hostname"}</code></nobr> | リッスンするホスト名                    |
| <nobr><code>{"--mdns"}</code></nobr>     | mDNS 検出を有効にする                   |
| <nobr><code>{"--cors"}</code></nobr>     | CORS を許可する追加のブラウザーオリジン |

### acp

oc2 acp

このコマンドは、nd-JSON を使用して stdin/stdout 経由で通信する ACP サーバーを起動します。

#### フラグ

| フラグ                                   | 説明                 |
| ---------------------------------------- | -------------------- |
| <nobr><code>{"--cwd"}</code></nobr>      | 作業ディレクトリ     |
| <nobr><code>{"--port"}</code></nobr>     | リッスンするポート   |
| <nobr><code>{"--hostname"}</code></nobr> | リッスンするホスト名 |

### uninstall

OC2 をアンインストールし、関連ファイルをすべて削除します。

```bash
oc2 uninstall
```

#### フラグ

| フラグ                                      | ショート | 説明                                         |
| ------------------------------------------- | -------- | -------------------------------------------- |
| <nobr><code>{"--keep-config"}</code></nobr> | `-c`     | 構成ファイルを保持する                       |
| <nobr><code>{"--keep-data"}</code></nobr>   | `-d`     | セッションデータとスナップショットを保持する |
| <nobr><code>{"--dry-run"}</code></nobr>     |          | 削除せずに削除される内容を表示する           |
| <nobr><code>{"--force"}</code></nobr>       | `-f`     | 確認プロンプトをスキップする                 |

---

### upgrade

oc2 を最新バージョンまたは特定のバージョンに更新します。

```bash
oc2 upgrade [target]
```

最新バージョンにアップグレードするには。

```bash
oc2 upgrade
```

特定のバージョンにアップグレードするには。

```bash
oc2 upgrade v0.1.48
```

#### フラグ

| フラグ                                 | ショート | 説明                                                   |
| -------------------------------------- | -------- | ------------------------------------------------------ |
| <nobr><code>{"--method"}</code></nobr> | `-m`     | 使用されたインストール方法。curl, npm, pnpm, bun, brew |

## 環境変数

OC2 は環境変数を使用して構成できます。

| 変数                                  | タイプ   | 説明                                                                |
| ------------------------------------- | -------- | ------------------------------------------------------------------- |
| `OC2_AUTO_SHARE`                 | ブール値 | セッションを自動的に共有する                                        |
| `OC2_GIT_BASH_PATH`              | 文字列   | Windows 上で実行可能な Git Bash へのパス                            |
| `OC2_CONFIG`                     | 文字列   | 構成ファイルへのパス                                                |
| `OC2_CONFIG_DIR`                 | 文字列   | config ディレクトリへのパス                                         |
| `OC2_CONFIG_CONTENT`             | 文字列   | インライン JSON 構成コンテンツ                                      |
| `OC2_DISABLE_AUTOUPDATE`         | ブール値 | 自動更新チェックを無効にする                                        |
| `OC2_DISABLE_PRUNE`              | ブール値 | 古いデータのプルーニングを無効にする                                |
| `OC2_DISABLE_TERMINAL_TITLE`     | ブール値 | ターミナルタイトルの自動更新を無効にする                            |
| `OC2_PERMISSION`                 | 文字列   | インライン化された json 権限設定                                    |
| `OC2_DISABLE_DEFAULT_PLUGINS`    | ブール値 | デフォルトのプラグインを無効にする                                  |
| `OC2_DISABLE_LSP_DOWNLOAD`       | ブール値 | LSP サーバーの自動ダウンロードを無効にする                          |
| `OC2_ENABLE_EXPERIMENTAL_MODELS` | ブール値 | 実験モデルを有効にする                                              |
| `OC2_DISABLE_AUTOCOMPACT`        | ブール値 | 自動コンテキスト圧縮を無効にする                                    |
| `OC2_DISABLE_CLAUDE_CODE`        | ブール値 | `.claude` からの読み取りを無効にする (プロンプト + スキル)          |
| `OC2_DISABLE_CLAUDE_CODE_PROMPT` | ブール値 | `~/.claude/CLAUDE.md` の読み取りを無効にする                        |
| `OC2_DISABLE_CLAUDE_CODE_SKILLS` | ブール値 | `.claude/skills` のロードを無効にする                               |
| `OC2_DISABLE_MODELS_FETCH`       | ブール値 | リモートソースからのモデルの取得を無効にする                        |
| `OC2_FAKE_VCS`                   | 文字列   | テスト目的の偽の VCS プロバイダー                                   |
| `OC2_CLIENT`                     | 文字列   | クライアント識別子 (デフォルトは `cli`)                             |
| `OC2_ENABLE_EXA`                 | ブール値 | Exa Web 検索ツールを有効にする                                      |
| `OC2_SERVER_PASSWORD`            | 文字列   | `serve`/`web` の基本認証を有効にする                                |
| `OC2_SERVER_USERNAME`            | 文字列   | 基本認証ユーザー名 (デフォルト `oc2`) をオーバーライドします。 |
| `OC2_MODELS_URL`                 | 文字列   | モデル設定を取得するためのカスタム URL                              |
