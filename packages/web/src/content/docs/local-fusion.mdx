---
title: Local Fusion
description: Fan out one prompt to multiple local model branches, judge the results, and synthesize a final answer.
---

Local Fusion is a compound workflow for moments when one model answer is not enough. It sends the same prompt to several configured branches, asks a judge model to compare the branch outputs, then has a synthesizer model produce one final response.

Branches and the judge are research stages. They should not edit workspace files; only the synthesizer should apply final workspace edits when its tools and permissions allow it.

That makes Local Fusion useful for code reviews, architecture tradeoffs, migration planning, or any task where you want independent perspectives before committing to an answer.

---

## Configure

Add a named `local_fusion` config to `oc2.json`.

```json title="oc2.json"
{
  "$schema": "https://oc2.ai/config.json",
  "local_fusion": {
    "review-panel": {
      "branches": [
        {
          "model": "anthropic/claude-sonnet-4-5",
          "variant": "default",
          "prompt": "Focus on implementation correctness, edge cases, and maintainability.",
          "toolPolicy": "readonly",
          "timeout": 120000
        },
        {
          "model": "openai/gpt-5",
          "variant": "high",
          "prompt": "Focus on product behavior, user impact, and missing tests.",
          "toolPolicy": "readonly",
          "timeout": 120000
        }
      ],
      "judge": {
        "model": "openai/gpt-5-mini",
        "prompt": "Rank the branch outputs by correctness, specificity, and risk coverage."
      },
      "synthesizer": {
        "model": "anthropic/claude-sonnet-4-5",
        "variant": "default",
        "prompt": "Combine the strongest findings into one concise answer with concrete next steps."
      },
      "limits": {
        "timeout": 180000,
        "maxBranches": 4
      }
    }
  }
}
```

Each branch can use a different model, variant, agent, prompt, tool policy, and timeout.

### Config Shape

Use this as a quick reference for named configs under `local_fusion`.

```jsonc title="oc2.jsonc"
{
  "local_fusion": {
    "config-name": {
      "branches": [
        {
          "model": "provider/model",
          "variant": "optional-variant",
          "agent": "optional-agent",
          "prompt": "Optional branch-specific guidance.",
          "toolPolicy": "readonly",
          "timeout": 120000,
        },
      ],
      "judge": {
        "model": "provider/model",
        "variant": "optional-variant",
        "prompt": "Optional judge guidance.",
        "toolPolicy": "none",
      },
      "synthesizer": {
        "model": "provider/model",
        "variant": "optional-variant",
        "prompt": "Optional synthesizer guidance.",
        "toolPolicy": "none",
      },
      "limits": {
        "timeout": 180000,
        "maxBranches": 3,
      },
    },
  },
}
```

`branches[].toolPolicy` defaults to `"readonly"`. `judge.toolPolicy` and `synthesizer.toolPolicy` default to `"none"`. `limits.maxBranches` defaults to `3`.

### Safe Review Mode

Use read-only branches when you want independent review without allowing branch stages to write anything.

### Scratch Research Mode

Use `all` or `parent_without_teams` on branches or the judge when they need temporary notes. Write-capable branch and judge stages are isolated to a per-stage temporary scratch directory outside all session roots.

```json title="oc2.json"
{
  "$schema": "https://oc2.ai/config.json",
  "local_fusion": {
    "scratch-research": {
      "branches": [
        {
          "model": "anthropic/claude-sonnet-4-5",
          "agent": "plan",
          "prompt": "Trace the code path. Use scratch files only for temporary notes.",
          "toolPolicy": "all"
        },
        {
          "model": "openai/gpt-5",
          "variant": "high",
          "agent": "plan",
          "prompt": "Independently analyze architecture tradeoffs and migration risks.",
          "toolPolicy": "parent_without_teams"
        }
      ],
      "judge": {
        "model": "openai/gpt-5-mini",
        "prompt": "Compare branch evidence and identify the strongest recommendation.",
        "toolPolicy": "all"
      },
      "synthesizer": {
        "model": "anthropic/claude-sonnet-4-5",
        "prompt": "Return the final answer only. Include concrete file paths and rationale.",
        "toolPolicy": "readonly"
      }
    }
  }
}
```

Scratch files are temporary-only artifacts, not workspace changes. Branch and judge stages can use `write` and `edit` only inside their assigned scratch directory; `apply_patch` and workspace edits are unavailable. OC2 places scratch directories outside all session roots. It starts from the system temp directory and falls back to a sibling temp base if the default scratch root would be inside a session root. Scratch directory retention is unspecified and not a cleanup guarantee.

### Implementation Mode

Use a write-capable synthesizer when Local Fusion should apply final workspace edits after branches and the judge finish. Keep branches focused on research and let only the synthesizer edit the workspace.

```json title="oc2.json"
{
  "$schema": "https://oc2.ai/config.json",
  "local_fusion": {
    "implementation-panel": {
      "branches": [
        {
          "model": "anthropic/claude-sonnet-4-5",
          "agent": "plan",
          "prompt": "Research the smallest correct implementation. Do not edit workspace files.",
          "toolPolicy": "readonly"
        },
        {
          "model": "openai/gpt-5",
          "variant": "high",
          "agent": "plan",
          "prompt": "Look for regressions, tests, and simpler alternatives. Do not edit workspace files.",
          "toolPolicy": "readonly"
        }
      ],
      "judge": {
        "model": "openai/gpt-5-mini",
        "prompt": "Select the safest implementation path and call out unresolved risks.",
        "toolPolicy": "none"
      },
      "synthesizer": {
        "model": "anthropic/claude-sonnet-4-5",
        "agent": "build",
        "prompt": "Implement only the selected final changes, then verify them when possible.",
        "toolPolicy": "all"
      }
    }
  }
}
```

For branches and the judge, write-capable policies are isolated to scratch. The synthesizer is the only stage that should apply final workspace edits.

## Tool Policy

`toolPolicy` controls tool access for Local Fusion child stages.

- `branches[].toolPolicy` defaults to `"readonly"`.
- `judge.toolPolicy` defaults to `"none"`.
- `synthesizer.toolPolicy` defaults to `"none"`.

Use `"readonly"` when a child stage should inspect files and search the workspace without making changes. Use `"none"` when a child stage should answer from the prompt and conversation context only.

| Policy                 | Branch                                                                                              | Judge                                                                                               | Synthesizer                                                                      |
| ---------------------- | --------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------- |
| `none`                 | No tools                                                                                            | No tools                                                                                            | No tools                                                                         |
| `readonly`             | Read, search, web, and LSP tools                                                                    | Read, search, web, and LSP tools                                                                    | Read, search, web, and LSP tools                                                 |
| `parent_without_teams` | Scratch-scoped `write` and `edit`; no workspace edits, `apply_patch`, teams, or nested Local Fusion | Scratch-scoped `write` and `edit`; no workspace edits, `apply_patch`, teams, or nested Local Fusion | Parent-style tools with `team_create`, `team_spawn`, and `local_fusion` disabled |
| `all`                  | Scratch-scoped `write` and `edit`; no workspace edits or `apply_patch`                              | Scratch-scoped `write` and `edit`; no workspace edits or `apply_patch`                              | Unrestricted tool map, still subject to parent and session permissions           |

`"parent_without_teams"` lets the synthesizer use the parent session's normal tool resolution, enables `task` subagents when the parent permissions allow them, and force-disables `team_create`, `team_spawn`, and `local_fusion` for that child stage. For branches and the judge, write-capable access is limited to a per-stage temporary scratch directory outside session roots; workspace edits and `apply_patch` remain denied. Child-spawned subagents also cannot create agent teams or invoke nested local fusion.

`"all"` gives the synthesizer the unrestricted tool map, still subject to parent and session permissions. For branches and the judge, it exposes research tools plus scratch-file tools for the per-stage temporary directory; `apply_patch` and workspace edits remain unavailable.

You can also set `toolPolicy` on inline `local_fusion` tool calls:

```json
{
  "prompt": "Review this change for correctness and simpler alternatives.",
  "branches": [
    {
      "model": "anthropic/claude-sonnet-4-5",
      "prompt": "Use repository context and return file-specific findings.",
      "toolPolicy": "readonly"
    },
    {
      "model": "openai/gpt-5",
      "prompt": "Do not use tools. Challenge the approach from first principles.",
      "toolPolicy": "none"
    }
  ],
  "judge": {
    "model": "openai/gpt-5-mini"
  },
  "synthesizer": {
    "model": "anthropic/claude-sonnet-4-5"
  }
}
```

## Failure Modes

- Invalid inline config fails before branches start.
- If every branch fails, the tool fails with branch failure summaries.
- If at least one branch succeeds, judge and synthesizer still run with the successful outputs and explicit failure summaries.
- Judge failure fails the tool; OC2 does not silently synthesize from raw branch outputs.
- Synthesizer failure fails the tool.
- Branch timeouts are recorded as branch failures, and the run continues if another branch succeeds.

## Run

Use the `/local_fusion` command with the config name followed by the prompt.

```bash frame="none"
/local_fusion review-panel Review the authentication refactor in this branch. Identify correctness risks, security regressions, missing tests, and any simpler implementation path. Return prioritized findings with file references and a short recommendation.
```

The first argument, `review-panel`, selects the named config. Everything after it becomes the prompt that is sent to each branch.

---

## When To Use It

Use Local Fusion when the cost of a shallow answer is higher than the cost of extra model calls. It is strongest when branches have different instructions, because each branch can pressure-test the problem from a different angle before the judge and synthesizer collapse the results into one answer.

For routine edits or small questions, a single model response is usually faster. For high-risk changes, Local Fusion gives you a lightweight review panel inside the same OC2 workflow.
