---
title: Local Fusion
description: Fan out one prompt to multiple local model branches, judge the results, and synthesize a final answer.
---

Local Fusion is a compound model workflow for moments when one model answer is not enough. It sends the same prompt to several configured branches, asks a judge model to compare the branch outputs, then has a synthesizer model produce one final response.

That makes it useful for code reviews, architecture tradeoffs, migration planning, or any task where you want independent perspectives before committing to an answer.

---

## Configure

Add a named `local_fusion` config to `opencode.json`.

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "local_fusion": {
    "review-panel": {
      "branches": [
        {
          "model": "anthropic/claude-sonnet-4-5",
          "variant": "default",
          "prompt": "Focus on implementation correctness, edge cases, and maintainability.",
          "toolPolicy": "readonly",
          "timeout": 120000
        },
        {
          "model": "openai/gpt-5",
          "variant": "high",
          "prompt": "Focus on product behavior, user impact, and missing tests.",
          "toolPolicy": "readonly",
          "timeout": 120000
        }
      ],
      "judge": {
        "model": "openai/gpt-5-mini",
        "prompt": "Rank the branch outputs by correctness, specificity, and risk coverage."
      },
      "synthesizer": {
        "model": "anthropic/claude-sonnet-4-5",
        "variant": "default",
        "prompt": "Combine the strongest findings into one concise answer with concrete next steps."
      },
      "limits": {
        "timeout": 180000,
        "maxBranches": 4
      }
    }
  }
}
```

Each branch can use a different model, variant, agent, prompt, tool policy, and timeout.

### Logu Local Model

`logu` is a local compound model exposed through the normal model picker as `logu/logu`. It is always selectable and does not require Logu provider credentials or a provider connect step, but it still needs the underlying branch, judge, and synthesizer providers configured.

Configure Logu with the reserved `local_fusion.logu` config name:

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "local_fusion": {
    "logu": {
      "branches": [
        {
          "model": "anthropic/claude-sonnet-4-5",
          "agent": "build",
          "toolPolicy": "parent_without_teams"
        },
        {
          "model": "openai/gpt-5",
          "agent": "plan",
          "toolPolicy": "parent_without_teams"
        }
      ],
      "judge": {
        "model": "openai/gpt-5-mini"
      },
      "synthesizer": {
        "model": "anthropic/claude-sonnet-4-5"
      },
      "limits": {
        "timeout": 120000,
        "maxBranches": 3
      }
    }
  }
}
```

When you select `logu/logu`, OpenCode renders the current conversation into a transcript, runs `local_fusion.logu`, and returns the synthesizer output as the assistant response in the parent session.

Do not use `logu/logu` inside `local_fusion.logu`. Branch, judge, and synthesizer models must reference real underlying provider models, such as `anthropic/claude-sonnet-4-5` or `openai/gpt-5`.

### Tool Policy

`toolPolicy` is configured per branch. Use `"readonly"` when a branch should inspect files and search the workspace without making changes. Use `"none"` when a branch should answer from the prompt and conversation context only.

Branches default to `"readonly"`, while the judge and synthesizer always run with tools disabled.

`"parent_without_teams"` is only supported by `local_fusion.logu`. It lets a Logu branch use the parent session's normal tool resolution, enables `task` subagents when the parent permissions allow them, and force-disables `team_create`, `team_spawn`, and `local_fusion` for the branch. Branch-spawned subagents also cannot create agent teams or invoke nested local fusion.

The `/local_fusion` command rejects `"parent_without_teams"`; use `"readonly"` or `"none"` for normal Local Fusion runs.

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "local_fusion": {
    "mixed-review": {
      "branches": [
        {
          "model": "anthropic/claude-sonnet-4-5",
          "prompt": "Inspect the code and identify concrete correctness risks.",
          "toolPolicy": "readonly"
        },
        {
          "model": "openai/gpt-5",
          "prompt": "Reason from the prompt only. Focus on architecture tradeoffs and missing assumptions.",
          "toolPolicy": "none"
        }
      ],
      "judge": {
        "model": "openai/gpt-5-mini"
      },
      "synthesizer": {
        "model": "anthropic/claude-sonnet-4-5"
      }
    }
  }
}
```

You can also set `toolPolicy` on inline `local_fusion` tool calls:

```json
{
  "prompt": "Review this change for correctness and simpler alternatives.",
  "branches": [
    {
      "model": "anthropic/claude-sonnet-4-5",
      "prompt": "Use repository context and return file-specific findings.",
      "toolPolicy": "readonly"
    },
    {
      "model": "openai/gpt-5",
      "prompt": "Do not use tools. Challenge the approach from first principles.",
      "toolPolicy": "none"
    }
  ],
  "judge": {
    "model": "openai/gpt-5-mini"
  },
  "synthesizer": {
    "model": "anthropic/claude-sonnet-4-5"
  }
}
```

## Failure Modes

If `logu/logu` is selected without `local_fusion.logu`, the assistant turn fails with:

```txt
logu requires local_fusion.logu config; see packages/web/src/content/docs/local-fusion.mdx
```

If a Logu config references `logu/logu` as an underlying model, the run fails before branch sessions are created. The error names the offending path:

```txt
logu config cannot reference logu/logu at branches[0].model
logu config cannot reference logu/logu at judge.model
logu config cannot reference logu/logu at synthesizer.model
```

A timed-out branch is recorded as a branch failure with `Branch timed out after <timeout>ms`. Logu continues when at least one branch succeeds.

If every branch fails, the parent assistant turn fails with `logu failed: All compound branches failed: ...`.

Logu is not available in V2 sessions in this first pass. It is only routed through the current in-process Local Fusion runtime.

## Run

Use the `/local_fusion` command with the config name followed by the prompt.

```bash frame="none"
/local_fusion review-panel Review the authentication refactor in this branch. Identify correctness risks, security regressions, missing tests, and any simpler implementation path. Return prioritized findings with file references and a short recommendation.
```

The first argument, `review-panel`, selects the named config. Everything after it becomes the prompt that is sent to each branch.

---

## When To Use It

Use Local Fusion when the cost of a shallow answer is higher than the cost of extra model calls. It is strongest when branches have different instructions, because each branch can pressure-test the problem from a different angle before the judge and synthesizer collapse the results into one answer.

For routine edits or small questions, a single model response is usually faster. For high-risk changes, Local Fusion gives you a lightweight review panel inside the same OpenCode workflow.
