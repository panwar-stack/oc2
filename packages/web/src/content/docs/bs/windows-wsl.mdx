Iako OC2 moze raditi direktno na Windowsu, preporucujemo [Windows Subsystem for Linux (WSL)](https://learn.microsoft.com/en-us/windows/wsl/install) za najbolje iskustvo. WSL daje Linux okruzenje koje glatko radi sa OC2 funkcijama.

:::tip[Zašto WSL?]
WSL nudi bolje performanse fajl sistema, punu terminalsku podrsku i kompatibilnost s razvojnim alatima na koje se OC2 oslanja.
:::

---

## Postavljanje

<Steps>

1. **Instalirajte WSL**

   Ako vec niste, [instalirajte WSL](https://learn.microsoft.com/en-us/windows/wsl/install) prema zvanicnom Microsoft vodicu.

2. **Instalirajte OC2 u WSL-u**

   Kad je WSL spreman, otvorite WSL terminal i instalirajte OC2 jednom od [metoda instalacije](/docs/).

   ```bash
   curl -fsSL https://oc2.ai/install | bash
   ```

3. **Koristite OC2 iz WSL-a**

   Idite u direktorij projekta (Windows fajlovima pristupate preko `/mnt/c/`, `/mnt/d/` itd.) i pokrenite OC2.

   ```bash
   cd /mnt/c/Users/YourName/project
   oc2
   ```

</Steps>

## Desktop aplikacija + WSL Server

Ako preferirate OC2 Desktop aplikaciju, ali zelite da server radi u WSL-u:

1. **Pokrenite server u WSL-u** sa `--hostname 0.0.0.0` da dozvolite vanjske konekcije:

   ```bash
   oc2 serve --hostname 0.0.0.0 --port 4096
   ```

2. **Povezite Desktop aplikaciju** na `http://localhost:4096`

:::note
Ako `localhost` ne radi u vasem setupu, povezte se preko WSL IP adrese (u WSL-u: `hostname -I`) i koristite `http://<wsl-ip>:4096`.
:::

:::caution
Kada koristite `--hostname 0.0.0.0`, postavite `OC2_SERVER_PASSWORD` da zastitite server.

```bash
OC2_SERVER_PASSWORD=your-password oc2 serve --hostname 0.0.0.0
```

:::

## Web klijent + WSL

Za najbolje web iskustvo na Windowsu:

1. **Pokrenite `oc2 web` u WSL terminalu** umjesto u PowerShell-u:

   ```bash
   oc2 web --hostname 0.0.0.0
   ```

2. **Otvorite iz Windows browsera** na `http://localhost:<port>` (OC2 ispisuje URL)

Pokretanje `oc2 web` iz WSL-a osigurava ispravan pristup fajl sistemu i terminalsku integraciju, a i dalje je dostupno iz Windows browsera.

## Pristup Windows fajlovima

WSL moze pristupiti svim Windows fajlovima kroz `/mnt/` direktorij:

- Disk `C:` → `/mnt/c/`
- Disk `D:` → `/mnt/d/`
- I tako dalje...

Primjer:

```bash
cd /mnt/c/Users/YourName/Documents/project
oc2
```

:::tip
Za najgladje iskustvo, razmislite da klonirate/kopirate repo u WSL fajl sistem (npr. pod `~/code/`) i tu pokrenete OC2.
:::

---

## Savjeti

- Drzite OC2 u WSL-u za projekte na Windows diskovima - pristup fajlovima je jednostavan
- Koristite VS Code [WSL ekstenziju](https://code.visualstudio.com/docs/remote/wsl) uz OC2 za integrisan tok rada
- Vase OC2 konfiguracije i sesije cuvaju se u WSL okruzenju na `~/.local/share/oc2/`
