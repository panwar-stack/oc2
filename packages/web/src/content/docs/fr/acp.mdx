OC2 prend en charge le [Agent Client Protocol](https://agentclientprotocol.com) (ACP), vous permettant de l'utiliser directement dans les éditeurs et IDE compatibles.

:::tip
Pour obtenir une liste des éditeurs et des outils prenant en charge ACP, consultez le [ACP progress report](https://zed.dev/blog/acp-progress-report#available-now).
:::

ACP est un protocole ouvert qui standardise la communication entre les éditeurs de code et les agents de codage AI.

---

## Configuration

Pour utiliser OC2 via ACP, configurez votre éditeur pour exécuter la commande `oc2 acp`.

La commande démarre OC2 en tant que sous-processus compatible ACP qui communique avec votre éditeur via JSON-RPC sur stdio.

Vous trouverez ci-dessous des exemples d'éditeurs populaires prenant en charge ACP.

---

### Zed

Ajoutez à votre configuration [Zed](https://zed.dev) (`~/.config/zed/settings.json`) :

```json title="~/.config/zed/settings.json"
{
  "agent_servers": {
    "OC2": {
      "command": "oc2",
      "args": ["acp"]
    }
  }
}
```

Pour l'ouvrir, utilisez l'action `agent: new thread` dans la **Palette de commandes**.

Vous pouvez également associer un raccourci clavier en modifiant votre `keymap.json` :

```json title="keymap.json"
[
  {
    "bindings": {
      "cmd-alt-o": [
        "agent::NewExternalAgentThread",
        {
          "agent": {
            "custom": {
              "name": "OC2",
              "command": {
                "command": "oc2",
                "args": ["acp"]
              }
            }
          }
        }
      ]
    }
  }
]
```

---

### IDE JetBrains

Ajoutez `acp.json` à votre [IDE JetBrains](https://www.jetbrains.com/) conformément à la [documentation](https://www.jetbrains.com/help/ai-assistant/acp.html) :

```json title="acp.json"
{
  "agent_servers": {
    "OC2": {
      "command": "/absolute/path/bin/oc2",
      "args": ["acp"]
    }
  }
}
```

Pour l'ouvrir, utilisez le nouvel agent « OC2 » dans le sélecteur d'agent AI Chat.

---

### Avante.nvim

Ajoutez à votre configuration [Avante.nvim](https://github.com/yetone/avante.nvim) :

```lua
{
  acp_providers = {
    ["oc2"] = {
      command = "oc2",
      args = { "acp" }
    }
  }
}
```

Si vous devez transmettre des variables d'environnement :

```lua {6-8}
{
  acp_providers = {
    ["oc2"] = {
      command = "oc2",
      args = { "acp" },
      env = {
        OC2_API_KEY = os.getenv("OC2_API_KEY")
      }
    }
  }
}
```

---

### CodeCompanion.nvim

Pour utiliser OC2 comme agent ACP dans [CodeCompanion.nvim](https://github.com/olimorris/codecompanion.nvim), ajoutez ce qui suit à votre configuration Neovim :

```lua
require("codecompanion").setup({
  interactions = {
    chat = {
      adapter = {
        name = "oc2",
        model = "claude-sonnet-4",
      },
    },
  },
})
```

Cette configuration configure CodeCompanion pour utiliser OC2 comme agent ACP pour le chat.

Si vous devez transmettre des variables d'environnement (comme `OC2_API_KEY`), reportez-vous à [Configuring Adapters: Environment Variables](https://codecompanion.olimorris.dev/getting-started#setting-an-api-key) dans la documentation CodeCompanion.nvim pour plus de détails.

## Prise en charge

OC2 fonctionne de la même manière via ACP que dans le terminal. Toutes les fonctionnalités sont prises en charge :

:::note
Certaines commandes slash intégrées telles que `/undo` et `/redo` ne sont actuellement pas prises en charge.
:::

- Outils intégrés (opérations sur les fichiers, commandes de terminal, etc.)
- Outils personnalisés et commandes slash
- Serveurs MCP configurés dans votre configuration OC2
- Règles spécifiques au projet à partir de `AGENTS.md`
- Formateurs et linters personnalisés
- Système d'agents et d'autorisations
