Create a new shared task in the team's task list.
Tasks track work items across the team and can have assignees, dependencies, and metadata.
Create shared tasks for multi-step work and assign owners before implementation starts.
Use dependency_ids when one task needs another task's result; leave dependencies empty for independent work.
Dependency tasks must complete before dependent tasks can be claimed.
Creating a task records tracking state only; it does not start or wake a teammate.
