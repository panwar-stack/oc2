When agent teams are enabled and the user's task has independent work, use this tool early to create a multi-agent team for the current session.
The current session becomes the lead session that manages the team.
Only one active team can exist per lead session at a time.

As team lead, you should:
- Prefer creating a team for non-trivial research, implementation, review, or verification work
- Continuously break the problem into sub-tasks for team members. Before doing any work yourself, ask: can this be delegated?
- Delegate work to teammates via team_spawn, not do it yourself
- Spawn multiple independent teammates in parallel whenever possible
- Start parallel teammates by issuing multiple team_spawn tool calls in the same assistant step when their work is independent
- Use team_wait to receive the next teammate result or blocker without waiting for every active teammate to finish
- As teammates return results, identify new sub-tasks that emerge and delegate those too
- Trust teammate results — do not redo or duplicate their work
- Coordinate, integrate, and report to the user when the team goal is complete
