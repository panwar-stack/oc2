Resolve and read $1 before creating any worktree.
Use agent teams.

Treat the checkout that invoked this command as read-only. Record its path and status, and record its committed HEAD as base_sha.
Create one unique implementation branch and one dedicated git worktree outside the invoking checkout, rooted at base_sha. Perform every edit, generation step, test, review, and commit from the isolated worktree.
Do not stash, reset, clean, switch, edit, merge into, or cherry-pick into the invoking checkout.
Keep uncommitted and untracked files in the invoking checkout, including the requested specification, as read-only inputs. Do not copy them into the implementation diff unless the selected pull request explicitly requires them. If implementation depends on other uncommitted source changes, stop and report that dependency.

If PR #$2 is provided, implement only that pull request from $1. Verify, test, review, and commit it.
If PR #$2 is missing, implement `$1` one pull request at a time in the same isolated worktree. For each pull request: implement, verify, test, review, commit. Start the next pull request only after the commit so each committed slice becomes the base for the next. At the very end, audit the whole implementation to ensure nothing was missed.

Do not implement unrelated pull requests, future phases, or adjacent improvements unless required for the current pull request.
Before the final response, verify that the invoking checkout's status still matches the recorded status.
On success or failure, preserve both checkouts and do not discard partial work automatically. Cleanup, merge, cherry-pick, or other integration requires a separate explicit request.
Report the isolated worktree path, branch, base_sha, created commits, verification results, and remaining worktree status.
