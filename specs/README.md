# Architecture Specifications

This directory is the single canonical tree for durable OC2 architecture, cross-module contracts, and implemented design decisions.

Specifications explain behavior that is difficult to recover from one source file. Source code remains authoritative for exact schemas, APIs, configuration fields, and implementation details.

Do not store implementation plans, migration checklists, progress reports, generated artifacts, product positioning, or unresolved proposals here. Track active work in issues and pull requests, keep contributor guidance in `AGENTS.md` or skills, and rely on Git history for removed plans.
